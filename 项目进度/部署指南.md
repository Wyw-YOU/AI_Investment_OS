## 2. 本地开发部署 (Win11 + PyCharm)

适用于本地开发调试，不依赖 Docker，直接运行后端和前端。

### 2.1 环境准备

**Python 环境** (后端):

```bash
# 确认 Python 版本 >= 3.11
python --version

# 进入后端目录
cd backend

# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境 (Win11 PowerShell)
.venv\Scripts\Activate.ps1

# 激活虚拟环境 (Git Bash)
source .venv/Scripts/activate

# 安装依赖 (使用腾讯云镜像加速)
pip install -r requirements.txt -i https://mirrors.tencent.com/pypi/simple/ --trusted-host mirrors.tencent.com
```

**Node.js 环境** (前端):

```bash
# 确认 Node.js 版本 >= 18
node --version

# 进入前端目录
cd frontend

# 设置 npm 腾讯云镜像 (推荐)
npm config set registry https://mirrors.tencent.com/npm/

# 安装依赖
npm install
```

**Redis** (可选，不装也能运行):

```bash
# 方式1: 使用 Docker 单独启动 Redis
docker run -d --name redis -p 6379:6379 redis:7-alpine

# 方式2: 不安装 Redis，系统会自动降级为内存缓存
```

### 2.2 配置环境变量

在**项目根目录**创建 `.env` 文件:

```bash
cd AI_Investment_OS
cp .env.example .env
```

编辑 `.env`:

```env
# LLM 配置 (必填)
LLM_API_KEY=your-deepseek-api-key
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat

# JWT 密钥 (必填，随便写一个随机字符串)
JWT_SECRET=my-local-dev-secret-123

# Redis (可选，不填则自动使用内存缓存)
REDIS_URL=redis://localhost:6379/0

# 数据库 (默认 SQLite，无需修改)
DATABASE_URL=sqlite+aiosqlite:///./data/investment.db

# CORS (本地开发前后端端口)
CORS_ORIGINS=http://localhost:3000,http://127.0.0.1:3000
```

### 2.3 启动后端

**终端 1** — 启动 FastAPI 后端:

```bash
cd backend

# 激活虚拟环境
.venv\Scripts\Activate.ps1   # PowerShell
# source .venv/Scripts/activate  # Git Bash

# 启动服务 (热重载模式)
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

看到以下输出表示成功:

```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
INFO:     Started reloader process
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.
```

验证: 浏览器打开 http://localhost:8000/docs 可看到 Swagger API 文档。

### 2.4 启动前端

**终端 2** — 启动 Next.js 前端:

```bash
cd frontend

# 启动开发服务器
npm run dev
```

看到以下输出表示成功:

```
  ▲ Next.js 14.2.x
  - Local:    http://localhost:3000
```

验证: 浏览器打开 http://localhost:3000 可看到 Dashboard 页面。

### 2.5 PyCharm 配置

**后端运行配置**:

1. PyCharm 打开 `backend/` 目录
2. `File > Settings > Project > Python Interpreter` → 选择 `.venv/Scripts/python.exe`
3. 创建 Run Configuration:
   - Script: `uvicorn`
   - Parameters: `app.main:app --reload --host 0.0.0.0 --port 8000`
   - Working directory: `backend/`
4. 勾选 `Emulate terminal in output console` (可选)

**测试运行配置**:

1. 创建 Run Configuration:
   - Script path: `tests/`
   - Working directory: `backend/`
   - 或直接在终端运行: `pytest tests/ -v`

### 2.6 本地验证全流程

```bash
# 1. 健康检查
curl http://localhost:8000/health
# 预期: {"status":"ok","version":"1.0.0"}

# 2. 注册用户
curl -X POST http://localhost:8000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"test@test.com","password":"test123456"}'
# 预期: {"code":0,"message":"success","data":{"user":...,"token":"..."}}

# 3. 获取热门股票 (无需登录)
curl http://localhost:8000/api/stocks/hot
# 预期: {"code":0,"data":[...]}

# 4. 触发 AI 分析 (需要登录 token)
# 用上面返回的 token 替换 <TOKEN>
curl -X POST http://localhost:8000/api/agent/run \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <TOKEN>" \
  -d '{"stock_code":"600519"}'
# 预期: 分析完成后返回完整报告 (约30-90秒)
```

### 2.7 常见本地问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `ModuleNotFoundError: No module named 'app'` | 未在 backend 目录运行 | `cd backend` 后再运行 uvicorn |
| `pip install` 报错 | 缺少 C 编译器 | 安装 Visual Studio Build Tools |
| Redis 连接失败 | 未启动 Redis | 系统自动降级为内存缓存，不影响运行 |
| AKShare 数据为空 | 非交易时间或网络问题 | 正常现象，系统会返回缓存数据或空数据 |
| LLM 调用超时 | API Key 错误或网络不通 | 检查 `.env` 中的 `LLM_API_KEY` 和 `LLM_BASE_URL` |
| 前端连不上后端 | CORS 或端口问题 | 确认后端在 8000 端口运行，检查 `.env` 中 CORS_ORIGINS |
| `langgraph` 安装失败 | 依赖冲突 | `pip install --upgrade pip` 后重试 |

### 2.8 一键启动脚本

项目根目录提供 `start-local.bat`，双击即可自动安装依赖并启动前后端:

```
AI_Investment_OS/
├── start-local.bat   <-- 双击运行
```

脚本会自动:
1. 检查 `.env` 文件是否存在
2. 检查 Python 和 Node.js 环境
3. 创建虚拟环境并安装后端依赖
4. 安装前端依赖 (首次)
5. 分别打开两个命令行窗口启动后端和前端

### 2.9 本地开发 vs Docker 对比

| 对比项 | 本地开发 | Docker 部署 |
|--------|----------|-------------|
| 适用场景 | 开发调试 | 生产/演示 |
| 启动方式 | 分别启动后端/前端 | `docker compose up -d` |
| 热重载 | 支持 (uvicorn --reload) | 需重新构建 |
| Redis | 可选 (自动降级) | 必须 (容器内置) |
| 数据库文件 | `backend/data/investment.db` | Docker volume 持久化 |
| 环境变量 | 项目根目录 `.env` | 项目根目录 `.env` |

## 3. 云服务器 Docker 部署

### 3.1 安装 Docker (Ubuntu)

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 添加用户到 docker 组
sudo usermod -aG docker $USER
newgrp docker

# 安装 Docker Compose (V2)
sudo apt install docker-compose-plugin -y

# 验证
docker --version
docker compose version
```

### 3.2 克隆代码

```bash
git clone <repository-url>
cd AI_Investment_OS
```

### 3.3 配置环境变量

```bash
cp .env.example .env
nano .env
```

填写以下必填项:

```env
LLM_API_KEY=your-deepseek-api-key
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
JWT_SECRET=$(openssl rand -hex 32)
```

### 3.4 启动服务

```bash
# 构建并启动
docker compose up -d --build

# 查看状态
docker compose ps

# 查看日志
docker compose logs -f backend
```

### 3.5 验证部署

## 5. 内存优化 (2C4G 环境)

### 已采用的优化策略

| 优化项 | 节省内存 | 说明 |
|--------|----------|------|
| SQLite 替代 PostgreSQL | ~300MB | 单文件数据库, 无需额外进程 |
| 不使用 Qdrant | ~500MB | MVP 阶段暂不使用向量检索 |
| 不使用 MinIO | ~200MB | 暂不使用对象存储 |
| Redis Alpine | ~50MB | 最小化 Redis 镜像 |
| **总计节省** | **~1050MB** | |

### 系统级优化

```bash
# 增加 swap (推荐)
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# 限制 Docker 日志大小
# 在 docker-compose.yml 每个服务下添加:
#   logging:
#     options:
#       max-size: "10m"
#       max-file: "3"
```

## 6. 反向代理 (Nginx)

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /api/ {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /ws/ {
        proxy_pass http://localhost:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## 9. 常见问题

### 9.1 内存不足 (OOM)

**症状**: 服务频繁重启
**解决**:
1. 增加 swap 空间 (2GB)
2. 重启 Docker: `docker compose restart`
3. 升级服务器到 4C8G

### 9.2 AKShare 数据获取失败

**症状**: 股票数据为空
**解决**:
1. 检查服务器网络: `curl https://push2.eastmoney.com`
2. 非交易时间部分数据可能不可用
3. 系统会自动使用缓存数据

### 9.3 LLM API 超时

**症状**: Agent 分析长时间无响应
**解决**:
1. 检查 API Key 是否正确
2. 确认网络可访问 LLM API 端点
3. 单个 Agent 超时不会中断整个工作流

### 9.4 端口被占用

**症状**: 服务启动失败
**解决**:
```bash
# 检查端口占用
sudo lsof -i :8000
sudo lsof -i :3000

# 修改 docker-compose.yml 端口映射
```
