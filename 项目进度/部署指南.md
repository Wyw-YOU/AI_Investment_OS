## 2. 部署步骤

### 2.1 安装 Docker

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 添加用户到 docker 组
sudo usermod -aG docker $USER
newgrp docker

# 安装 Docker Compose (V2)
sudo apt install docker-compose-plugin -y

# 验证
docker --version
docker compose version
```

### 2.2 克隆代码

```bash
git clone <repository-url>
cd AI_Investment_OS
```

### 2.3 配置环境变量

```bash
cp .env.example .env
nano .env
```

填写以下必填项:

```env
LLM_API_KEY=your-deepseek-api-key
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
JWT_SECRET=$(openssl rand -hex 32)
```

### 2.4 启动服务

```bash
# 构建并启动
docker compose up -d --build

# 查看状态
docker compose ps

# 查看日志
docker compose logs -f backend
```

### 2.5 验证部署

```bash
# 健康检查
curl http://localhost:8000/health

# 前端访问
curl -I http://localhost:3000

# 测试 API
curl -X POST http://localhost:8000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@test.com","password":"admin123456"}'
```

## 4. 内存优化 (2C4G 环境)

### 已采用的优化策略

| 优化项 | 节省内存 | 说明 |
|--------|----------|------|
| SQLite 替代 PostgreSQL | ~300MB | 单文件数据库, 无需额外进程 |
| 不使用 Qdrant | ~500MB | MVP 阶段暂不使用向量检索 |
| 不使用 MinIO | ~200MB | 暂不使用对象存储 |
| Redis Alpine | ~50MB | 最小化 Redis 镜像 |
| **总计节省** | **~1050MB** | |

### 系统级优化

```bash
# 增加 swap (推荐)
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# 限制 Docker 日志大小
# 在 docker-compose.yml 每个服务下添加:
#   logging:
#     options:
#       max-size: "10m"
#       max-file: "3"
```

## 5. 反向代理 (Nginx)

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /api/ {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /ws/ {
        proxy_pass http://localhost:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## 8. 常见问题

### 8.1 内存不足 (OOM)

**症状**: 服务频繁重启
**解决**:
1. 增加 swap 空间 (2GB)
2. 重启 Docker: `docker compose restart`
3. 升级服务器到 4C8G

### 8.2 AKShare 数据获取失败

**症状**: 股票数据为空
**解决**:
1. 检查服务器网络: `curl https://push2.eastmoney.com`
2. 非交易时间部分数据可能不可用
3. 系统会自动使用缓存数据

### 8.3 LLM API 超时

**症状**: Agent 分析长时间无响应
**解决**:
1. 检查 API Key 是否正确
2. 确认网络可访问 LLM API 端点
3. 单个 Agent 超时不会中断整个工作流

### 8.4 端口被占用

**症状**: 服务启动失败
**解决**:
```bash
# 检查端口占用
sudo lsof -i :8000
sudo lsof -i :3000

# 修改 docker-compose.yml 端口映射
```
