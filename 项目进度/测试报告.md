## 3. Agent 工作流测试

### 3.1 完整分析流程

```
输入: stock_code="600519", stock_name="贵州茅台"

预期执行流程:
Planner -> [News, Financial, Technical, Macro] (并行) -> merge -> Risk -> Quant -> Report

输出:
- phase: "complete"
- final_report: {executive_summary, recommendation, overall_score, ...}
- risk_assessment: {overall_risk, risk_score}
- quant_score: {composite_score, rating}
- agent_outputs: 8 个 Agent 的结果
```

### 3.2 错误降级测试

| 场景 | 预期行为 |
|------|----------|
| 单个 Agent LLM 调用失败 | 返回 confidence=0.0 的降级结果, 其他 Agent 继续 |
| AKShare 数据获取超时 | 服务返回空数据, Agent 使用已有上下文 |
| 所有并行 Agent 失败 | Risk/Quant/Report 仍可运行, 使用空输入 |
