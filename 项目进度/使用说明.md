# AI Investment OS - 使用说明

---

## 1. 快速开始

### 1.1 克隆项目

```bash
git clone <repository-url>
cd AI_Investment_OS
```

### 1.2 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env` 文件:

```env
# DeepSeek 配置 (推荐)
LLM_API_KEY=your-deepseek-api-key
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat

# 或 Xiaomi MiMo
# LLM_API_KEY=your-mimo-api-key
# LLM_BASE_URL=https://api.xiaomi.com/v1
# LLM_MODEL=mimo-chat

JWT_SECRET=your-random-secret-key
```

### 1.3 Docker 一键启动

```bash
docker-compose up -d
```

### 1.4 访问系统

- 前端: http://localhost:3000
- 后端 API: http://localhost:8000
- API 文档: http://localhost:8000/docs

## 3. 功能说明

### 3.1 Dashboard (首页)

- **市场概览**: 上证/深证/创业板指数
- **AI 今日机会**: 推荐分析入口
- **Agent 系统状态**: 8 个 Agent 的运行状态
- **热门股票**: 按成交额排序的热门 A 股

### 3.2 股票分析

1. 在顶部输入框输入股票代码 (如 `600519`)
2. 进入股票详情页, 查看:
   - **K 线图**: 蜡烛图 + MA5/MA10/MA20 + 成交量 + MACD
   - **价格信息**: 实时价格、涨跌幅、开盘/最高/最低
3. 点击 "AI 深度分析" 按钮
4. 等待 Agent 工作流完成 (约 30-90 秒)
5. 查看分析结果:
   - 投资报告 (摘要 + 推荐 + 评分)
   - 核心发现
   - 风险评估
   - 各 Agent 置信度

### 3.3 Portfolio (研究空间)

- 创建研究空间 (输入股票代码)
- 跟踪关注的股票
- 点击进入股票分析页

### 3.4 登录/注册

- 邮箱 + 密码注册
- JWT Token 自动管理
- 未登录时部分功能受限

## 4. AI Agent 分析流程

```
用户输入股票代码
    |
    v
数据收集 (AKShare)
├── 实时行情
├── K 线历史
├── 财务数据
└── 新闻数据
    |
    v
Planner Agent (任务规划)
    |
    v (并行执行)
├── News Agent       -- 新闻情感分析
├── Financial Agent  -- 基本面分析
├── Technical Agent  -- 技术面分析
└── Macro Agent      -- 宏观环境分析
    |
    v (合并)
Risk Agent  -- 风险综合评估
    |
    v
Quant Agent -- 量化评分 (0-100)
    |
    v
Report Agent -- 生成投资报告
    |
    v
展示结果
```

## 5. LLM 配置

### DeepSeek (推荐, 性价比高)

```env
LLM_API_KEY=sk-xxx
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
```

### Xiaomi MiMo

```env
LLM_API_KEY=xxx
LLM_BASE_URL=https://api.xiaomi.com/v1
LLM_MODEL=mimo-chat
```

### 切换 LLM

修改 `.env` 中的 `LLM_BASE_URL` 和 `LLM_MODEL`, 重启后端即可。系统使用 OpenAI 兼容协议, 支持任何兼容 API。

## 7. 常见问题

**Q: 分析失败怎么办?**
A: 检查 LLM API Key 是否正确, 网络是否通畅。单个 Agent 失败不会中断整个流程。

**Q: 如何更换 LLM 提供商?**
A: 修改 `.env` 中的 `LLM_BASE_URL` 和 `LLM_MODEL`, 重启后端。

**Q: 股票数据获取失败?**
A: AKShare 可能受网络影响, 系统会自动使用缓存数据。检查服务器网络连接。
